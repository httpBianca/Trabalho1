Para quem é a acessibilidade?

A acessibilidade na web é voltada para pessoas com algum tipo de deficiência, seja visual, auditiva, motora ou cognitiva. Por exemplo, pessoas cegas ou com baixa visão, daltônicos, pessoas surdas ou com perda auditiva, pessoas com dislexia, autismo, síndrome de Down ou Parkinson, entre outras condições. Foram desenvolvidas tecnologias assistivas para proporcionar maior independência e autonomia a esses usuários, como leitores de tela, ampliadores de tela, mouses e teclados especiais. Essas ferramentas, no entanto, serão melhor aproveitadas se os códigos do site seguirem determinados padrões.

Links

Os links do tipo LEIA MAIS não são suficientemente descritivos fora do contexto. Exemplo:

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. Para acessa-lo clique aqui.

Textos de links devem ser descritivos, Exemplo correto, com o link em toda a frase, sem o “Leia mais”:

GitHub é uma plataforma de hospedagem de código-fonte e arquivos com controle de versão usando o Git. GitHub.

Imagens:

A tag img possui um atributo chamado alt que permite especificar um texto alternativo que descreve a imagem. Este texto é lido por softwares leitores de tela e também é reconhecido por mecanismos de busca. A falta do atributo alt em imagens é uma das falhas mais comuns em acessibilidade web.
